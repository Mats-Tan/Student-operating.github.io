如何在 Gitee 上用 hexo 搭建博客

我的电脑环境：

OS：win10

博客环境安装与配置

我使用的工具

  • git bash

    使用目的：使用其自带的 git 工具

  • nood.js

    使用目的：

    1. hexo 是基于 nood.js 环境的博客环境，必须要安装 nood.js
    2. 使用 nood.js 中的 npm 工具

  • npm

    使用目的：使用 npm 工具下载 cnpm 工具

  • cnpm

    使用目的：下载 hexo 环境配置

  • hexo

    使用目的：快速生成 hexo 博客环境

  • Typora

    使用目的：写博客用的 md 文件编辑器

工具准备

Git Bash

Git Bash 是一个类似于 Linux 的终端，常见的操作是用来项目管理，可以向网络上的仓库上传和拉取内容，也可以建立分支进行管理。

下载地址：https://gitforwindows.org/

我已经安装过了所以会显示这个界面，没有安装过的同学不断 next 就可以。

Typora

Typora 是一个轻量化的 md 文件编辑器，支持 markdown 语法， 能够便捷的进行文件的书写和样式美化

下载地址：https://www.typora.net/

使用 Typora 需要学习 markdown 语法 ，这里推荐一个很好的学习地址，只需要 数 10 分钟就能基本掌握 markdown 语法

教程地址：https://markdown.com.cn/basic-syntax/

nood.js

nood.js 选择于自己操作系统相同的安装包进行安装。

下载地址：http://nodejs.cn/download/

右击桌面上的 “此电脑” ，查看 “属性” 可以知道自己电脑的系统类型

安装结束后，打开 git （或 cmd ），输入：node -v 检查安装是否成功

cmd

  1. win+R

  2. 输入 cmd

  3. 输入 node -v

    如果这里显示的没有类似于 v14.17.6 这样的版本号，则安装失败

  4. 输入 npm -v

    如果没有类似图片这样的版本号，则安装失败

git

  1. 任意处右击鼠标（或双击 git 打开方式）

  1. 输入 node -v 和 npm -v

    结果和 cmd 是一样的

cnpm

cnpm 属于 npm 在中国的一个拓展，因为在国内使用 npm 访问失败率高。

安装 cnpm

任意打开 cmd （或 git bash），输入以下内容：

npm install -g cnpm --registry=https://registry.npm.taobao.org

输入结束后回车等待，当显示类似于以下消息：

C:\xxx\xxx\npm\cnpm -> C:\xxx\xxx\bin\cnpm ，表示安装成功。

hexo

hexo 是一种简洁的博客框架，使用 markdown 语法生成 静态网页。

安装 hexo
  1. 使用 cnpm 安装 hexo

命名：cnpm install -g hexo-cli

（图片转载自：CSDN：jiuqi_玖柒）

  1. 验证是否安装成功

    hexo -v

    这是成功后的版本信息

  2. 初始化 hexo

    1. 创建一个空文件夹
    2. 进入文件夹右击鼠标，打开 git bash
    3. 输入：hexo init
    4. 成功后文件夹中会生成很多文件

  3. 运行 hexo

    命令：hexo s ——启动本地博客

    命令：CTRL+C ——停止本地博客

    启动本地博客后可以在浏览器地址栏输入 localhost:4000 预览

  4. hexo 常用命令

    1. 创建新文章

      hexo new "title" 双引号内的内容是我的文章标题，可以随意修改。

      创建成功的文章会保存在：/source/_posts/

      文件的格式是 .md

      文章可以用 typora 进行编辑

    2. 清除博客缓存

      命令：hexo clean

      使用场景：切换博客主题等需要对全局配置进行修改时

    3. 将 .md 文件生成静态页面

      命令：hexo g

      使用场景：完成 .md 文件的编辑后

  1. 在 Gitee 创建仓库

    1. 进入 Gitee，地址：https://gitee.com/

      1. 仓库名称要填写自己的用户名

        如果用其他名称，那么无法用自己的仓库名直接访问，需要通过用户名间接访问

      2. 开源

      3. 设置模板

  2. 安装 hexo-deployer-git 插件，用于设置远程仓库

    命令：cnpm install --save hexo-deployer-git

  3. 设置远程仓库

    在文件中找到 _config.yml 做出以下修改：

    仓库的名字与用户名一致

    注意：type:后一定要留空格，这里会监测缩进

  4. 部署到远端的 Gitee

    1. 部署到远端的码云仓库

      命令：hexo d

  5. 开启静态页面服务：Gitee Pages

设置图床

图床的作用是什么呢？

​ 我在写博客的过程中带有图片是十分常见的，但是图片的体积很大，如果都存储在博客的框架资源中，会很占用我每次的上传和加载运行，所以我选择使用图床。

​ 虽然听起来是个新鲜的名词，但其实就像我在前面的操作，将代码上传到 Gitee 一样，我也可以将图片上传的 Gitee 来存储，每次访问界面的图片就可以来自于网络了，当然如果付费可以选择阿里、七牛等存储效果更好的地址。

​ 接下来让我展示一下，如何用 PicGo + Typora 的形式来进行博客图片的实时上传存储。

写一篇博客并上传的流程

写博客

两种方式

  1. hexo new "name"
  2. 在文件夹 source/_posts/ 中添加新的 .md 文件

清除缓存

  1. 执行 hexo clean 清除缓存
